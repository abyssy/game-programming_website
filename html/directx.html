Birth


The first version of DirectX was released in September 1995 as the Windows Games SDK.

It was the Win32 replacement for the DCI and WinG APIs for Windows 3.1.

DirectX allowed all versions of Microsoft Windows, starting with Windows 95, to incorporate high-performance multimedia.

In late 1994, Microsoft was ready to release Windows 95, its next operating system.

DOS allowed direct access to video cards, keyboards, mice, sound devices, and all other parts of the system, while Windows 95 - with its protected memory model - restricted access to all of these, working on a much more standardized model.

Eisler, St. John, and Engstrom worked together to fix this problem, with a solution that they eventually named DirectX.




Feature


DirectX is multifunctional.

OpenGL is just a graphical library. But DirectX includes graphics, sound, input, network and other modules.

Graphics part is the key of processing graphics, divided into DirectDraw and Direct3D. The most important API in the sound section is DirectSound.

DirectInput can support a lot of game input devices, it can make these devices fully play the best state and all the features.

DirectPlay is mainly for the network function of the game and the support of a variety of connections, TCP / IP, IPX, Modem, serial port, etc., so that players can use a variety of networking methods to fight.




Use


DirectX is not a simple graphical API, it is a widely used API that developed by Microsoft's, which contains Direct Graphics, Direct Input, Direct Play, Direct Sound, Direct Show, Direct Setup, Direct Media Objects and other components, it provides a set of multimedia interface program.

At first, DirectX is developed to make up for the Windows 3.1 system on the graphics, voice processing capabilities, but now it has developed into a interface that has a decisive impact on all aspects of the entire multimedia system.
